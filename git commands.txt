Track changes: Git diff: 
Track the changes that have not been staged:

Track the changes between two commits:

Stashing Changes : 
Temporarily saves changes without committing them:
git stash
Restoring Stashed Changes : Restores the most recent stashed changes:
git stash pop
Displays only the names of stashed commits:
git stash show
Shows a list of all stashes with indices (stash@{0}, stash@{1}, etc.):
git stash list
Removes the most recent stash:
git stash drop
Deletes all stashes:
git stash clear














The three file stages of Git are:
1. Committed
The unmodified changes from the last snapshot.
2. Modified
The files changed or modified; You could edit, add or delete contents of an existing file or could totally add or delete files.
3. Staged
The files bookmarked to be added to the next commit snapshot.  

Every File changed in the project folder can be:
Tracked by git OR Untracked by git

Process:
1. created TOC.txt
Added a line: Hello
2. git status : showed us an untracked file : TOC.txt
3. git add TOC.txt 
4. git Status: now TOC.txt is staged (tracked), but not committed
5. modified TOC.txt
Added one more line: World
6. git status : TOC.txt is staged ; TOC.txt is modified post add
Right now if we commit the working directory, we will get the old added file of TOC.txt (with only Hello). Why? Because we used git add before modification. 
It means that only staged files are committed
7. If you want the modified file to be committed, pl add it to the staged area via the add command
